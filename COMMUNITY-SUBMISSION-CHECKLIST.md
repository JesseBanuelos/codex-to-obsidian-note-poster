# Community Submission Checklist

- [ ] Test the plugin in a real desktop Obsidian vault
- [ ] Confirm the local server binds only to `127.0.0.1`
- [ ] Set a non-empty API key and verify unauthorized requests fail
- [ ] Build fresh release assets with `npm run build`
- [ ] Verify `manifest.json`, `main.js`, and `styles.css` are ready for release upload
- [ ] Update `versions.json` for the release version
- [ ] Create a public GitHub release for the plugin
- [ ] Submit the plugin for Obsidian community review
